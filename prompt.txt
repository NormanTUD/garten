## Architektur-Prinzipien

1. **Ultra-modular**: Viele kleine Dateien, klare Ordnerstruktur, Single Responsibility
2. **API-First**: Alles lГӨuft Гјber die REST-API, Frontend ist ein reiner Client
3. **Jeder API-Zugriff wird geloggt** (User, Endpoint, Timestamp, Request/Response)
4. **VollstГӨndig testbar**: Unit Tests, Integration Tests, E2E Tests
5. **Backup-freundlich**: DB-Dump Export/Import per API und CLI
6. **Erweiterbar**: Datenmodelle sind flexibel (dynamische Felder/Tags wo sinnvoll)
7. **Code-Sprache**: Englisch (Variablen, Kommentare, API-Endpunkte, Git Commits)
8. **UI-Sprache**: Deutsch

## Datenmodell (Kern-Entities)

### Users & Auth
- **User**: id, username, password_hash, display_name, role (admin/user),
  created_at, is_active
- **APIKey**: id, user_id, key_hash, label, created_at, last_used, is_active
- **AuditLog**: id, user_id, method, endpoint, request_body, response_status,
  ip_address, timestamp

### Gartenstruktur
- **Garden**: id, name, description, location (lat/lng), total_area_sqm
- **Bed**: id, garden_id, name, description, geometry (GeoJSON Polygon fГјr
  Leaflet-Karte), area_sqm, soil_type, sun_exposure, created_at
- **Plant**: id, name, variety, category, icon, expected_water_needs,
  growing_season_start, growing_season_end, notes
  (Pflanzen-Katalog der sich Гјber die Zeit fГјllt)
- **BedPlanting**: id, bed_id, plant_id, planted_at, expected_harvest,
  status (active/harvested/removed), notes

### Ernte-Tracking
- **Harvest**: id, user_id, bed_id, plant_id, amount, unit (kg/stГјck/bund/...),
  quality_rating, date, notes, created_at
- **HarvestPhoto**: id, harvest_id, image_path, ai_detection_result, created_at

### BewГӨsserung
- **WateringEvent**: id, user_id, bed_id, started_at (exakte Zeit),
  duration_minutes, water_amount_liters, method (manual/automatic),
  weather_temp_c, weather_humidity_pct, weather_rain_mm,
  weather_description, soil_moisture_before, soil_moisture_after, notes
- **FertilizingEvent**: id, user_id, bed_id, fertilizer_type, amount,
  unit, date, notes

### Finanzen
- **ExpenseCategory**: id, name, icon, is_active (dynamisch erweiterbar)
- **Expense**: id, user_id, category_id, amount_cents, description, date,
  is_recurring, recurrence_interval, receipt_image_path,
  ocr_raw_result, created_at
- **ExpenseSplit**: id, expense_id, user_id, share_amount_cents, is_settled
- **Payment**: id, from_user_id, to_user_id, amount_cents, method
  (cash/transfer/material), description, date, confirmed_by_admin

### Gartenstunden (Vereinsarbeit)
- **GardenDutyConfig**: id, year, required_hours_per_person,
  hourly_rate_cents (Ausgleichszahlung)
- **GardenDutyLog**: id, user_id, date, hours, description, confirmed_by (admin),
  created_at
- **GardenDutyBalance**: (computed view) user_id, year, hours_done,
  hours_required, hours_remaining, compensation_due_cents

### Sensor-Vorbereitung (fГјr spГӨtere BewГӨsserungs-KI)
- **Sensor**: id, bed_id, type (soil_moisture/temperature/light/rain),
  hardware_id, is_active, last_reading_at
- **SensorReading**: id, sensor_id, value, unit, timestamp
- **IrrigationZone**: id, name, beds (m2m), valve_hardware_id, is_active
- **IrrigationSchedule**: id, zone_id, trigger_type (manual/scheduled/ai),
  cron_expression, ai_model_version, is_active
